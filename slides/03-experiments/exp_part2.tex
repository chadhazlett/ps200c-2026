\begin{frame}{Covariates}
\small
\begin{itemize}
\makegaps
\item<1-> Randomization is gold standard for causal inference because in expectation it balances \alert{observed} but also \alert{unobserved} characteristics between treatment and control group.\medskip
\item<2->  Unlike potential outcomes, you observe baseline covariates for all units. 
\item<3-> We mean ``pre-treatment'' covariates here,  or rather, those not affected by $D_i$ ...\pause \textcolor{Blue}{Though there are some other interesting cases of covariates you should not include,  but that's for another day.}
\item<4->  Randomization implies \textit{covariate balance},  $$p(X|D=1)=p(X|D=0)$$
\item<5-> If this is not the case, then one of two possibilities: 
\begin{itemize}
%  \itemsep1pt\parskip0pt\parsep0pt
  \item randomization was compromised.
  \item finite sample bad luck
\end{itemize}
\item<6-> Good idea to (i) check ``balance'', and/or (ii) adjust for chance imbalances to at least improve efficiency.
\end{itemize}
\end{frame}

\begin{frame}{Regression with Covariates}
\small
\onslide<1-> A common model with experimental data:
\[ Y_i = \alpha + \tau D_i + X_i\beta + \epsilon_i \]
\onslide<2-> Though soon we will consider this (better) model:
\[ Y_i = \alpha + \tau D_i + (X_i-\overline{X})\beta + D_i (X_i-\overline{X}) \theta +  \epsilon_i \]

\onslide<3->  But why include $X_i$ when experiments ``control'' for covariates by design?\\
\footnotesize
 \begin{itemize}
  \itemsep1pt\parskip0pt\parsep0pt 
  \item<4-> Correct chance imbalances that may push $\hat{\tau}$ far from $\tau_{ATE}$.
    \smallskip
	\item<5-> Increase precision
	\smallskip
	\item<6-> (Those are really the same in a given sample)
\end{itemize}
\small
\medskip
\onslide<7-> ATE estimates are robust to model specification (with sufficient $N$).\\
\begin{itemize}
\footnotesize
\item<8-> But, do not control for \textbf{post-treatment} covariates!\
\item<9-> This is an over-simplification: sometimes post-tx variables are okay, sometimes your QOI involves post-tx control, and sometimes pre-tx variables are problematic...we will need DAGs for that. 
\end{itemize}

\begin{frame}{Covariate Adjustment with Regression}
\small
Freedman (2008) noted that regression of the form:
\[ Y_i = \alpha + \tau_{reg} D_i + \beta_1 X_i  + \epsilon_i \]\\
is slightly biased (order $1/N$) in this scenario, does not necessarily improve precision, creates incorrect confidence intervals.\\
\bigskip
%\begin{itemize}
%\footnotesize
%\itemsep1pt\parskip5pt\parsep0pt
%\item<1-> $\hat{\tau}_{reg}$; small bias on order of $1/N$
%\item<2-> $\hat{\tau}_{reg}$ will not necessarily improve precision, confidence intervals wrong.
%\end{itemize}
\onslide<3-> Lin (2013) argued:\\
\begin{itemize}
\itemsep1pt\parskip5pt\parsep0pt
\footnotesize
\item<4-> if you interact covariates with $D$ it never hurts and likely improves precision:
\[ Y_i = \alpha + \tau_{Lin} D_i + \beta_1 \cdot (X_i - \bar X) + \beta_2 \cdot D_i \ \cdot (X_i - \bar X)  + \epsilon_i \] \\
\item<5-> this is what we mimicked graphically
\end{itemize}
\small
\onslide<5-> Better still, (see \href{https://doi.org/10.48550/arXiv.2403.03299}{this working paper}):
\begin{itemize}
\footnotesize
\item<6-> as we discuss later, regression with covariates induces a ``weird weighting'' we will discuss later... turns out that this interactive regression avoids this 
\item<7-> this is also equivalent to some other methods you may hear about (g-computation, t-learner, regression imputation)
%\item<8-> interactions not needed in large sample, but might as well
\end{itemize}
\end{frame}

\begin{frame}{Why the weird demeaning business in this model?}
\onslide<1-> It is isomorphic to a model without the demeaning, but makes interpretation for ATE more immediate.\\
 
\[ Y_i = \alpha + \tau_{Lin} D_i + \beta_1 \cdot (X_i - \bar X) + \beta_2 \cdot D_i \ \cdot (X_i - \bar X)  + \epsilon_i \]

%\onslide<2-> Review:
%\begin{itemize}
%\footnotesize
%\item What is $\frac{\partial Y_i}{\partial X_i}$? $\frac{\partial Y_i}{\partial D_i}$?
%\end{itemize}
\medskip
\onslide<2-> Interpreting $\tau_{Lin}$: 
\begin{itemize}
\footnotesize
\item<3-> What is $\E[Y_i|D_i=0, X_i=\overline{X}]$?
\item<4-> What is $\E[Y_i|D_i=1, X_i=\overline{X}]$?
\item<5-> What is $\E[Y_i|D_i=1,X_i=\overline{X}]-\E[Y_i|D_i=0, X_i=\overline{X}]$?
\item<6-> What is $\E[Y_i|D_i=1,X_i=x^{\ast}]-\E[Y_i|D_i=0, X_i=x^{\ast}]$ for $x^{\ast} \neq \overline{X}$?
\end{itemize}

\onslide<7-> So, 
\begin{itemize}
\footnotesize
\item<7-> $\tau_{Lin}$ tells you comparison after ``moving'' all points to  $X_i=\overline{X}$
%\item<8-> same as $\E_X[ \hat{\tau}|X=x]$, so convenient (faster: what is $\E_X[\frac{\partial Y_i}{\partial D_i}]$?)
\item<9-> model could be used to give you $ATE(X)$ but conveniently, $\E_x[ATE(X)]=\tau_{Lin}$. 
\item<10-> allowing this interaction is a good idea, emerging as standard practice
\end{itemize}
\end{frame}

\begin{frame}{Experiments are robust to alternative specifications}
\onslide<1-> Two facts:
\footnotesize
\begin{enumerate}
\item<1-> \textit{any} bivariate regression of some $Y$ on $X$ has $\hat{\beta}=\frac{\widehat{\Cov}(Y,X)}{\widehat{\Var}(X)}$. 
\item<2-> the coefficient on the $k$th variable, $X_k$, from a multivariate regression can be obtained by (i) regress $X_k$ on other $X$'s, call residual $\tilde{X}_k$.(ii) regress $Y$ on $\tilde{X}_k$. (Or $\tilde{Y}$ on $\tilde{X}_k$ for similarly defined $\tilde{Y}$)
\end{enumerate}

\onslide<3-> All together, this gives the Frisch-Waugh-Lovell (FWL) theorem:
\[\beta_k = \frac{Cov(\tilde{Y_i}, \tilde{X}_{ki})}{Var(\tilde{X}_{ki})}\] 

Where,  
\onslide<3->
\footnotesize
\begin{itemize}
\item $\tilde{X}_{ki}$ is the residual from a regression of $X_{ki}$ on all other covariates
\item $\tilde{Y_i}$ may have all other $X$ partialled out as well or you can use the original $Y_i$.  
\end{itemize}
\bigskip
\footnotesize
\end{frame}

\begin{frame}{Why are Experimental Findings Robust to Alternative Specifications?}
\small
\onslide<1-> If our ``$X_k$'' of interest is the $D$:
\[\hat{\beta}_D = \frac{\widehat{\Cov}(\tilde{Y_i}, \tilde{D}_i)}{\widehat{\Var}(\tilde{D}_{i})}\] 
where $\tilde{D}_i$ is the residual part of $D_i$ after regressing on $X_i$ \\
\medskip
\footnotesize
\begin{itemize}
\item<2-> For experimental data, on average, what will $\tilde{D}_i$ look like? \\
\item<3-> What does this suggest about the effect of adding covariates to the simple regression of $Y$ on $D$?
\end{itemize}
\end{frame}

\begin{frame}{Summary: Covariate Adjustment with Regression}
\small
\begin{itemize}
\itemsep1pt\parskip0pt\parsep0pt
\item<1-> ``Agnostic'' view of regression: one does not need to believe in the classical linear model (linearity and constant treatment effects) to advocate OLS covariate adjustment in randomized experiments \medskip
\item<2-> Helpful both to protect against chance imbalance and improve efficiency.\medskip
\item<3-> Since covariates are balanced by design, results  typically not model dependent \medskip
\item<4-> Best reason to be skeptical of adjustment: fishing.
\begin{itemize}
\footnotesize
\item always report unadjusted result first
\item best if adjustment strategy pre-specified
\end{itemize} 
\end{itemize}
\end{frame}

\subsection*{Blocking}

\begin{frame}{Blocking}
\small
\begin{itemize}
\itemsep1pt\parskip5pt\parsep0pt
\item<1->  Prior to randomization, collect background data on each unit. 
\item<2-> Why leave it to chance to balance on these, especially in small sample?
\item<3-> Blocking: better-than random balance by design:
\begin{itemize}
\item pre-stratify sample on important factor(s)
\item randomize within each stratum
\end{itemize}
\item<4-> Randomization is still there to balance unobserved attributes in expectation
\item<5-> Why is this helpful?
  \begin{itemize}
  \itemsep1pt\parskip0pt\parsep0pt
\footnotesize  

  \item<5-> Four subjects with $Y_{0i}$ given by $\{2,2,8,8\}$. (Say $Y_{1i}$ are similar). 
  \item<6->   Simple random assignment into two treated/control will place $\{2,2\}$ and $\{8,8\}$ together in the same group one-third of the time
  \end{itemize}
\end{itemize}
\end{frame}

\begin{frame}{Blocking}
\small
\onslide<1-> Imagine you run an experiment where you block on sex as designated at birth. \\
\medskip
\onslide<2-> Obtain block specific ATEs, $\tau_{f}=\E[\tau | f]$, $\tau_{m}=\E[\tau | m]$.  \\
\medskip
\medskip
\onslide<3-> Applying the LIE,
\[ \E [\tau] = p[f]\E[\tau|f]  + p[m]\E[\tau|m]\]

\onslide<5->  The sample analog is unbiased for this:
\[ \hat \tau_{B} = \frac{N_f}{N_f + N_m} \cdot \hat \tau_f  + \frac{N_m}{N_f + N_m} \cdot \hat \tau_m\]
\onslide<6->
or more generally, if there are $J$ strata or blocks, then
\[\hat\tau_B = \sum_{j=1}^J \frac{N_j}{N} \hat\tau_j\]
\end{frame}

\begin{frame}{Blocking}
\small
\onslide<1-> Taking the number of units per block as fixed (i.e. conditional on the experimental design), the variance is:
\onslide<2->
\footnotesize
\[\Var(\hat \tau_B) = \left (\frac{N_f}{N_f + N_m} \right)^2 \Var(\hat \tau_f) + \left(\frac{N_m}{N_f + N_m} \right)^2 \Var(\hat \tau_m)\]
\bigskip
\small
\onslide<3-> or more generally
\footnotesize
\[ Var(\hat \tau_B)  =  \sum_{j=1}^J  \left (\frac{N_j}{N} \right)^2  \Var(\hat \tau_j)\]
\end{frame}


\begin{frame}{Blocking with Regression}
\small
\onslide<1-> Can we treat these blocks as regressors in OLS?  \onslide<2-> \textcolor{Blue}{Yes, with caution} \\

\medskip

\onslide<2-> If Pr(D) is equal in all blocks, can use OLS with block intercepts (fixed effects):

\[ Y_i =  \beta_0 +  \tau D_i + \sum_{j=2}^{J} \beta_j \cdot B_{j[i]} + \epsilon_i\]

%where $B_j[i]$ is a dummy for the $j$-th block (one omitted as reference category).\bigskip \pause

\medskip
\onslide<3-> If (empirical) $Pr(D=1)$ changes by block, OLS differently weights the blocks.\\
\medskip
\onslide<4-> We can correct for this by going back to the Lin-style (interacted) regression!'
\footnotesize
\begin{itemize}
\item Again see \href{https://doi.org/10.48550/arXiv.2403.03299}{this working paper}
\item Turns out to be a better approach here than demeaning, but that is in the weeds.
\item Another proposal is to weight each observation so that block-treatment probabilities become the same. Let
\begin{align*}
p_{ij} &\equiv Pr(D_i=1|\text{ block for unit i}) \\
w_{ij} &= \left ( \frac{1}{p_{ij}} \right ) D_i + \left ( \frac{1}{1 - p_{ij}} \right ) (1 - D_i)
\end{align*}
...which turns out to be exactly the same thing as the interactive regression
\end{itemize}
\end{frame}


\begin{frame}{When does blocking in the analysis help?}
\small Imagine models for complete and block-randomized designs:

\begin{small}
\begin{eqnarray}
  Y_i &=& \alpha + \tau_{CR} D_i + \textcolor{red}{\varepsilon_i} \pause \\
  Y_i &=& \alpha + \tau_{BR} D_i + \sum_{j=2}^J \beta_j B_{j[i]}+ \textcolor{blue}{\varepsilon^{\ast}_i}
\end{eqnarray}
\end{small}

where $B_j[i]$ is a dummy for the $j$-th block. Then given iid sampling:
\pause
\small
\begin{eqnarray}
Var[\widehat{\tau}_{CR}]  &= \frac{\textcolor{red}{\sigma_{\varepsilon}^2}}{\sum_{i=1}^n (D_i - \bar{D})^2}\quad \,\,\, \quad \quad \mbox{ with } \textcolor{red}{\widehat{\sigma}_{\varepsilon}^2} \quad = \pause \quad\frac{\sum_{i=1}^n \widehat{\varepsilon}_i^2}{n-2}=\frac{SSR_{\widehat{\varepsilon}}}{n-2}\nonumber \\
\pause Var[\widehat{\tau}_{BR}]  &=  \frac{\textcolor{blue}{\sigma_{\varepsilon^{\ast}}^2}}{\sum_{i=1}^n (D_{i} - \bar{D})^2 (1-R_D^2)}\mbox{ with } \textcolor{blue}{\widehat{\sigma}_{\varepsilon^{\ast}}}^2 = \frac{\sum_{i=1}^n \widehat{ \varepsilon }^{\ast^2}_{i}}{n-k-1}=\frac{SSR_{\widehat{\varepsilon^{\ast}}}}{n-k-1}\nonumber
\end{eqnarray}
\normalsize
\pause
where $R_D^2$ is $R^2$ from regression of $D$ on all block indicators and a constant.
\end{frame}

\begin{frame}{When does blocking in the analysis help?}
\begin{eqnarray}
Var[\widehat{\tau}_{CR}]  &= \frac{\textcolor{red}{\sigma_{\varepsilon}^2}}{\sum_{i=1}^n (D_i - \bar{D})^2}\quad \,\,\, \quad \quad \mbox{ with } \textcolor{red}{\widehat{\sigma}_{\varepsilon}^2} \quad = \quad\frac{\sum_{i=1}^n \widehat{\varepsilon}_i^2}{n-2}=\frac{SSR_{\widehat{\varepsilon}}}{n-2}\nonumber \\
 Var[\widehat{\tau}_{BR}]  &=  \frac{\textcolor{blue}{\sigma_{\varepsilon^{\ast}}^2}}{\sum_{i=1}^n (D_{i} - \bar{D})^2 (1-R_j^2)}\mbox{ with } \textcolor{blue}{\widehat{\sigma}_{\varepsilon^{\ast}}^2} = \frac{\sum_{i=1}^n \widehat{ \varepsilon }^{\ast^2}_{i}}{n-k-1}=\frac{SSR_{\widehat{\varepsilon^{\ast}}}}{n-k-1}\nonumber
\end{eqnarray}
\footnotesize
where $R_D^2$ is $R^2$ from regression of $D$ on all $B$
variables and a constant.\\
\normalsize
\bigskip
\onslide<2->
\small
\begin{itemize}
\item<3-> If $R_D^2$ is high, variance inflated.  But what can we say about $R_D^2$ when we're talking about blocking?
\item<4-> If blocks explain outcomes strongly, how will $\textcolor{blue}{\widehat{\sigma}_{\varepsilon^{\ast}}^2}$ 
compare to $\textcolor{red}{\widehat{\sigma}_{\varepsilon}^2}$?
\item<5-> Adding DOF adjustment, we end up asking whether $\frac{SSR_{\widehat{\varepsilon^{\ast}}}}{n-k-1}<\frac{SSR_{\widehat{\varepsilon}}}{n-2}$
\end{itemize}
\end{frame}

\begin{frame}{Conclusions: Blocking}
\small
\onslide<1->  How does blocking help? \\
\footnotesize  
  \begin{itemize}
  \itemsep1pt\parskip0pt\parsep0pt
  \item Increases efficiency if the blocking variables predict outcomes 
 \item By design, reduced finite-sample bias due to ``bad'' randomization
 \item Powerful -- can save your experiment! \pause
   \end{itemize}
   \bigskip
   \small
\onslide<2->  What to block on?
\footnotesize
  \begin{itemize}
  \itemsep1pt\parskip0pt\parsep0pt
  \item Two constraints: data you can get, number of units to work with. 
  \item Irrelevant variables: burn up DOFs%,  worsen matches on good ones
  \item Most important: predictors of potential outcomes!    \item Pre-Tx outcomes is great;
add other main predictors if available/helpful
  \item Another consideration: variables desired for subgroup analysis
  \end{itemize}
  \small
\bigskip
\onslide<3-> How to block? \\
\footnotesize  
  \begin{itemize}
  \itemsep1pt\parskip0pt\parsep0pt
  \item Stratification
  \item Pair-wise matching; two pair-matching
%  \item Check: \texttt{blockTools} library.
  \end{itemize}
\end{frame}


\begin{frame}{Analysis with Blocking}
\small
\onslide<1-> \textbf{``As ye randomize, so shall ye analyze''} (Senn 2004)

\begin{itemize}
\itemsep1pt\parskip5pt\parsep0pt
\item<2-> Account for the method of randomization when doing  analysis

\item<3-> If using OLS, 
\begin{itemize} 
\item<3-> include block dummies 
\item<3-> typically use clustered SEs
\item<3-> if probability of treatment assignment varies across blocks, reweight
\end{itemize}
\item<4-> If doing randomization inference, block design influences construction of $\Omega$: re-randomization is within blocks.
\item<5-> Also, type of SE can influence type of  bootstrap, e.g. use the block bootstrap (resample at level of blocks) for clustered SEs
%\item<6-> Failure to account for the method of randomization in these ways can result in incorrect test size.
\end{itemize}
\end{frame}

\begin{frame}{Conclusion: Experiments}
\small
\begin{itemize}
\item<1-> Random assignment is great, but can use a helping hand sometimes...
\smallskip
\item<2-> Design: blocking to improve balance
\begin{itemize}
\item<2-> improves balance
\item<2-> improves efficiency
\item<2-> can save your experiments in small samples
\end{itemize}
\smallskip
\item<3-> Analysis: covariate adjustment
\begin{itemize}
\item<4-> can improve efficiency and finite sample bias
\item<4-> relatively safe, use interactive model (Lin 2013)
\item<4-> main risk is user-defined specification (fishing)
\begin{itemize}
\item<5-> report unadjusted results first
\item<5-> commit to adjustment plan, use non-arbitrary  choices
\item<5-> typically results will stable though
\end{itemize}
\end{itemize}
\smallskip
\item<7->  Beyond methodology, remember to consider:
\begin{itemize}
\item<8-> can your experiment reveal an interesting effect?
\item<8-> will it have external validity 
\item<8-> ethical concerns
\end{itemize}
\end{itemize}
\end{frame}
